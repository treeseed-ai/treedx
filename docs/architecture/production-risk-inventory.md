# TreeDX Production Risk Inventory

Status: Unified production risk inventory  
Source: TreeDX architecture, security, operations, and release-gate reviews

| Risk | Impact | Owner area | Implementation area | Current mitigation | Required implementation item | Verification test |
|---|---|---|---|---|---|---|
| JWKS/control-plane auth is under-specified | Production auth weakness | API auth | Auth and policy | Verifier abstraction, JWKS/OIDC verifier, cache/rotation behavior, and fail-closed boot config are implemented | Keep live issuer/key-rotation checks in deployment validation | `apps/api/test/treedx/auth_verifier_test.exs` |
| Direct exec sandbox leaks host data | Critical security issue | API exec/runtime | Git/exec/storage hardening + production hardening | `direct_dev` is explicit and rejected in connected/prod unless overridden; `container_sandbox` disables network and applies resource limits; `external_worker` and `firecracker_or_microvm` route through a signed worker boundary | Add live operator worker conformance tests in release environments | `apps/api/test/treedx_web/exec_sandbox_test.exs` |
| Federation leaks hidden data through ranking/counts | Cross-tenant data leak | Federation/query/graph | Federation execution + production hardening | Global federation now executes only reduced authorized repo/ref/path scopes, merges after filtering, sanitizes partial failures, and has live-gated multi-node checks | Add full distributed tracing in observability work | `apps/api/test/treedx_web/federated_*_test.exs`, `apps/api/test/treedx_web/leakage_regression_test.exs`, `scripts/acceptance/federation-live-check.sh` |
| Storage corruption loses metadata | Data loss | Rust store/native API | Auth and policy, Git/exec/storage hardening, production hardening | Append-only `.tdb` records, checksums, data-dir lock, recursive check, latest-record compaction, logical backup, migration metadata, guarded restore verification, and restore apply gate | Add external backup provider conformance and restore drills in release governance | `crates/treedx_store/tests/recovery_tests.rs`, `apps/api/test/treedx_web/admin_storage_controller_test.exs` |
| Binary files cannot move through no-clone APIs | SDK users fall back to local clones or leak binary data through text responses | API files/store/SDK | Binary transport + production hardening | Git blob reads, workspace blob APIs, raw upload/download, multipart upload sessions, and snapshot artifacts are byte-safe | Add multipart interoperability tests against hosted deployments | `apps/api/test/treedx_web/blob_controller_test.exs`, `crates/treedx_store/tests/workspace_blob_tests.rs`, `packages/ts-sdk/test/utils/treedx-blobs.test.ts` |
| SDK remote mode breaks local SDK ergonomics | Adoption blocker | TypeScript SDK | SDK remote integration | Local mode remains default, TreeDX is opt-in, and no-clone mode is explicit | Keep exports stable, add no-clone AgentSdk wiring, typed ports, and local-vs-remote parity tests | `packages/ts-sdk/test/utils/treedx-remote-mode.test.ts`, `packages/ts-sdk/test/utils/treedx-adapters.test.ts` |
| Graph/search/context diagnostics leak hidden data | Cross-tenant data leak | Graph/search/context | Graph/search/context productionization | Federation execution reduces scope before execution | Generate ranking and context diagnostics only from authorized results and logical paths | `apps/api/test/treedx_web/search_index_controller_test.exs`, `apps/api/test/treedx_web/context_modes_test.exs` |
| Graph refresh is too expensive or stale | Slow or incorrect graph results | Graph/search/context | Graph/search/context productionization | Full refresh exists and graph manifests persist | Add changed-path refresh metadata, stale base detection, refresh job records, and safe fallback to full refresh | `apps/api/test/treedx_web/graph_incremental_refresh_test.exs`, `crates/treedx_store/tests/graph_refresh_job_tests.rs` |
| Search indexing drifts from repository authorization | Hidden path leakage or stale search | Search/query | Graph/search/context productionization | Direct repository scan filters by capability before ranking | Add search index manifests/segments and status/compaction APIs that store only authorized logical metadata | `crates/treedx_store/tests/search_index_tests.rs`, `apps/api/test/treedx_web/search_index_controller_test.exs` |
| API/server and SDK type drift | Runtime failures | API contract/SDK | API contract and SDK remote integration | OpenAPI is schema-backed, SDK TreeDX API types are generated, and contract tests validate route/schema/request drift | Keep generated types fresh and extend contract tests for new routes | `treedx-exports.test.ts`, `treedx-contract-drift.test.ts`, `treedx-generated-types.test.ts` |
| Git push leaks credentials | Secret exposure | Git/mirror/audit | Git/exec/storage hardening + production hardening | Push rejects credential-bearing URLs, redacts file/local remotes, and resolves authenticated HTTPS/SSH through logical credential IDs and opt-in external transport | Add hosted credential-provider conformance tests | `crates/treedx_git/tests/push_tests.rs`, `apps/api/test/treedx_web/push_controller_test.exs` |
| Policy revocation does not affect active workspaces | Unauthorized access persists | Policy/workspaces | Auth and policy | Workspace policy hash/version checks and quarantine paths are implemented | Keep revocation/quarantine regression tests in release verification | `apps/api/test/treedx_web/workspace_revocation_test.exs` |
| Product semantics creep into TreeDX | Architecture degradation | API/storage/SDK boundary | All surfaces | Docs state TreeDX is repository scoped | Keep TreeSeed model/product concepts in SDK/core/control plane | Architecture review and API/schema grep |
| Shell Git fallback becomes default | Reliability/security drift | Git integration | Git/exec/storage hardening + production hardening | Push path remains Rust/local-file by default; external Git transport is disabled unless explicitly configured and uses scrubbed env plus credential IDs | Replace constrained external transport with native `gix` HTTPS/SSH push when upstream support is suitable | `crates/treedx_git/tests/push_tests.rs`, `apps/api/test/treedx_web/push_controller_test.exs` |
| Operational signals leak sensitive context | Secret or tenant metadata exposure | API observability | Observability and operations | Public health and metrics are sanitized; production logs use a scrubber-backed JSON formatter | Keep metrics label allowlists bounded and add log/metrics hygiene tests for new operational fields | `apps/api/test/treedx/observability/*_test.exs`, `apps/api/test/treedx_web/metrics_controller_test.exs` |
